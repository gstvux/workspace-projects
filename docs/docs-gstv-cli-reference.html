Guia de Instalação e Referência de Comandos

Instruções práticas para configurar, registrar e estender o CLI de automação multiprojeto gstv.

1. Configuração do Arquivo `project.yaml`

Para definir o nome de invocação do CLI no terminal, edite ou crie o arquivo project.yaml no diretório raiz do projeto: 

# project.yaml
project_name: "meu-projeto"
cli_command: "gstv"     # Nome da palavra-chave global para este projeto (ex: gstv, vela, teia)

2. Registro Global e Alinhamento de PATH

O CLI registra links simbólicos automaticamente no diretório do usuário ~/.local/bin.

Passo 1: Registrar o comando localmente

Execute o script roteador a partir da raiz do projeto:

./gstv relink

Passo 2: Garantir ~/.local/bin no PATH do Shell

Se for alertado de que o diretório não está no seu PATH, acrescente ao seu arquivo de perfil do shell (ex: ~/.bashrc ou ~/.zshrc): 

export PATH="$HOME/.local/bin:$PATH"

Recarregue o terminal usando: source ~/.bashrc

3. Comandos de Sistema do Roteador

Comando Descrição Operacional Escopo
relink Lê o project.yaml, remove qualquer link antigo que aponte para este projeto em ~/.local/bin/ (Garbage Collection) e cria o novo atalho atualizado. Sistema / Roteador
-h, --help Apresenta a tela de ajuda listando os metadados do projeto ativo e gerando dinamicamente a tabela de scripts executáveis da pasta /scripts. Ajuda / Metadados

4. Scripts Disponibilizados no Projeto

Estes scripts residem no diretório scripts/ e são disparados pelo comando principal:

serve-docs Executável

Inicia um servidor web local via Python HTTP server para renderizar toda a documentação sob a pasta docs/ e dispara o navegador silenciosamente. 

gstv serve-docs

5. Guia para Escrever Novos Scripts Portáveis

Para estender o catálogo de automação de qualquer projeto usando este template, basta criar um arquivo dentro de scripts/ seguindo estas regras fundamentais:

  • Sem Extensão: Crie o script sem a terminação .sh (ex: crie scripts/deploy).
  • Cabeçalho de Descrição: Adicione o comentário # description: ... na segunda linha do arquivo.
  • LF & Executável: Salve o script com quebras de linha padrão UNIX (LF) e execute chmod +x scripts/nome-do-script.

Template Exemplo (scripts/meu-script):

#!/usr/bin/env bash
# description: Executa tarefas de testes e formatação do repositório de forma automatizada

# Resolve caminhos relativos ao script de forma robusta e dinâmica
REAL_PATH="$(readlink -f "${BASH_SOURCE[0]}")"
SCRIPT_DIR="$(cd "$(dirname "$REAL_PATH")" && pwd)"
PROJECT_ROOT="$(dirname "$SCRIPT_DIR")"

echo -e "\033[1;32mIniciando execução do script...\033[0m"
# Insira sua automação aqui
